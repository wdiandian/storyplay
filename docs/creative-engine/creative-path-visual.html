StoryPlay 创作路径可视化

当前 StoryPlay 更接近实时故事生成器：玩家给一个故事命题，Creative Engine 通过内部 Agent System 生成一幕可玩的视觉故事。后续 Creator System 应该编辑产品层配置，而不是直接编辑 agent / skill。

1. 当前最短链路

Input

用户输入

故事想法、主角倾向、画风、节奏等首页配置。

Runtime

Session

保存 worldSetting、styleGuide、history、characters、storyState。

Agents

directScene

串联 Writer、CharacterDesigner、Cinematographer、Painter。

Output

Scene / Beat

返回可播放场景、节拍、对白、选择、图片和故事记忆。

Loop

玩家推进

选择、自由输入、背景点击会进入下一轮生成或探索。

2. Agent System 位置

TEXT

Writer

生成 plan、story、choices 和 storyState 更新。

TEXT

StyleSelector

在 auto 画风时选择内置视觉风格。

TEXT / IMAGE / TTS

CharacterDesigner

生成角色卡、肖像和音色。

TEXT

Cinematographer

把剧情意图转为英文镜头画面 prompt。

IMAGE

Painter

合并画风、镜头、角色参考图，生成场景图。

VISION

Vision

理解玩家点击的背景区域。

TEXT

FreeformClassifier

分类自由输入的意图。

TEXT

InsertBeat

在当前场景内插入临时节拍。

TTS

Voice / TTS

准备角色音色并合成对白音频。

当前治理口径是 9 个注册 agent 节点；agent / skill 是内部基建，不给创作者直接编辑。

3. 模型 API 对应关系

模型角色 使用节点 维护重点
TEXT_* Writer、StyleSelector、CharacterDesigner、Cinematographer、FreeformClassifier、InsertBeat 按 agent 验证 schema、parser、fallback 和内容质量。
IMAGE_* CharacterDesigner、Painter 验证参考图优先级、风格一致性、失败降级。
VISION_* Vision 验证点击区域理解和可交互建议。
TTS_* CharacterDesigner、Voice / TTS 验证音色分配、对白合成和静音 fallback。

模型升级不应全局替换后直接上线，应按 agent 节点分别跑 fixture / golden eval。

4. Creator System 边界

Creator

创作者编辑

World Config、Character Config、Narrative Rules、Visual Style Config、Interaction Rules。

Creative Engine

产品配置转运行时

把创作者配置转换为 Session、StoryState、Scene、Beat 等运行时结构。

Internal

Agent System

维护 contract、skill、prompt、parser、fallback、runtime、fixtures。

创作者不能直接编辑底层 agent contract、skill、parser、fallback 或 provider fallback。
