# getdesign.md Reference Index

This folder stores curated reference notes only. Do not copy whole external
DESIGN.md files into the product repo unless a specific implementation task
requires a short excerpt.

Source: https://getdesign.md/

## Storage Rule

Keep reference artifacts small and interpretive:

- Store links and storyplay adaptation notes.
- Store short implementation decisions.
- Do not vendor full website text, screenshots, or large kits into this repo.
- When implementing UI, read [../design-system.md](../design-system.md) first and use
  this index only as background.
