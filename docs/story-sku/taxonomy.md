# Story SKU Taxonomy

Status: MVP Draft.

本文档定义 StoryPlay 的第一版 SKU 分类体系。它用于首页分发、故事列表筛选、推荐策略、创作者发布表单和 `/studio/skus` 管理台。

## 设计原则

- `tags` 保留为轻量补充标签，不再承担全部分类职责。
- 结构化字段用于产品分发和筛选，必须可控、可枚举。
- SKU 分类描述的是消费侧体验，不描述完整创作工程。
- `First Act` 内容不在 SKU 分类里编辑，只作为 runtime 入口审计。

## 字段

### 主类型 `genres`

描述故事最主要的题材，可多选，建议 1-3 个。

- 恋爱
- 悬疑
- 奇幻
- 都市
- 校园
- 职场
- 科幻
- 历史
- 冒险
- 成长

### 情绪调性 `moods`

描述用户进入故事后的核心情绪预期，可多选，建议 1-4 个。

- 甜
- 虐
- 紧张
- 治愈
- 暧昧
- 爽感
- 暗黑
- 浪漫
- 诡异

### 互动强度 `interaction`

描述用户选择对体验的影响程度，单选。

- 轻互动：选择主要影响语气、节奏或局部反馈。
- 中互动：选择影响关系、线索或后续段落。
- 强互动：选择明显改变剧情走向、结局或状态。

### 叙事结构 `structure`

描述故事的主要推进方式，单选。

- 单线推进
- 多分支
- 短剧反转
- 关系推进
- 解谜
- 养成

### 视觉风格 `visualStyle`

描述首图、封面和后续素材的主视觉方向，单选。

- 电影感
- 轻小说
- 二次元
- 写实
- 哥特
- 赛博朋克
- 国风
- 波普

### 内容提示 `contentWarnings`

描述可能影响用户预期的内容，可多选。

- 惊悚
- 暴力
- 血腥
- 权谋
- 情感压迫
- 身份欺骗

## 和旧标签的关系

当前预设故事来自元项目，原始 `tags` 中混合了题材、情绪、结构和风格。MVP 阶段会保留 `tagsText` 作为 legacy tags，同时通过 `inferStorySkuTaxonomy` 推断第一版结构化分类。

后续正式发布链路里，创作者应直接选择结构化分类；legacy tags 只作为搜索补充或迁移字段。
